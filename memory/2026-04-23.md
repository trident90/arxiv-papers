# 2026-04-23 - Rate Limit 이슈 해결

## 문제 상황

- 2026-04-22에 188개 논문이 게시되었음
- API 요청 시 429 Rate Limit 오류 발생
- 대량 테스트로 인한 IP 일시적 차단

## 해결책

### ✅ 스크립트 개선 (Commit: 0222e42)

**요청 분산 전략:**
1. 각 카테고리당 3번의 배치 요청
   - Batch 1: 0-100
   - Batch 2: 100-200
   - Batch 3: 200-300

2. **대기 시간 증가**
   - 배치 간 대기: 3초
   - 카테고리 간 대기: 4초
   - 총 예상 수집 시간: ~5분

3. **429 오류 감시**
   - Rate limit 감지 시 즉시 중단
   - 불필요한 요청 방지

### 📝 변경 사항

**Before:**
```python
max_results: 500  # 한 번에 대량 요청
time.sleep(2.0)   # 짧은 대기
```

**After:**
```python
# 3개 배치 × 100개씩 분산 요청
for batch in range(3):
    start: batch * 100
    max_results: 100
    time.sleep(3.0)  # 배치 간 대기
time.sleep(4.0)     # 카테고리 간 대기
```
